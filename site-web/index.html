Héberger un site statique avec Docker

Cette page illustre un projet minimal pour servir un site web au sein d'un conteneur Docker.

Démarrer Voir le conteneur

Pourquoi Docker ?

Docker garantit un environnement isolé, reproductible et portable. Idéal pour déployer un site statique rapidement.

Ce dépôt

Il contient un Dockerfile, cette page index.html et une feuille de styles style.ss.

Serveur web

Vous pouvez utiliser Nginx, Caddy ou Apache pour servir les fichiers statiques à l'intérieur du conteneur.

Lancer localement avec Docker

  1. Construire l'image
  2. Démarrer le conteneur en exposant le port HTTP
  3. Ouvrir le navigateur et vérifier
docker build -t site-docker-demo .
docker run --rm -p 8080:80 --name site-web site-docker-demo

# Ouvrir le navigateur : http://localhost:8080

Astuce : adaptez le Dockerfile selon le serveur choisi et le chemin des assets.

Structure minimale

  • Dockerfile
  • index.html
  • style.ss

Bonnes pratiques

  • Utiliser des images officielles légères (e.g. nginx:alpine).
  • Copier uniquement les fichiers nécessaires.
  • Exposer le port requis et définir la commande par défaut.

Validation

Une fois lancé, surveillez les logs du conteneur et testez l'accessibilité depuis votre réseau si nécessaire.
